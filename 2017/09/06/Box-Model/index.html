盒子模型

在HTML文档中，每个元素都是一个矩形的盒子，而每个盒子的渲染由他们的尺寸、属性、位置等决定。

结构

一个独立的盒子模型由四个部分组成，分别为content、padding、border以及margin。如下图（截自ChromeDevTool）：

盒子大小

  • 宽度 = width + padding-left + padding-right + border-left + border-right；
  • 高度 = height + padding-top + padding-bottom + border-top + border-bottom。

注：在怪异模式，或者元素设置为box-sizing:border-box时

  • 宽度 = width ( = padding-left + padding-right + contentWidth)；
  • 高度 = height ( = padding-top + padding-bottom + contentHeight)；

块级盒的默认宽度

如果没有定义宽度，该元素的width并不是100%，而是”whatever is left”，即100%父元素（或浏览器窗口）宽度减去margin和padding后所剩的宽度。其中，textare标签中的col属性会影响默认样式，所以通常需要明确设置为100%。

绝对定位或浮动盒子的默认宽度、高度

没有设置宽度的绝对定位的盒子，他们的宽度只需要保持内容的宽度就足够了，如果框中包含一个单词，那么这个框只会呈现出单词的宽度，盒子默认宽度会随内容的增加而增加，直到为父类宽度的100%。默认高度则依据内容高度。浮动元素的盒子默认宽高同理。

行内元素的盒子模型

行内元素可以看做是很长很细的长方形，同样可以有margin``padding``border，但是尽管内容周围存在内边距与边框，但其占用空间（行高）由 line-height 属性决定。

关于border-box

box-sizing:content-box|border-box|inherit;
box-sizing:border-box;在IE8+以及其他主流浏览器中得到支持，IE7及以下版本无法识别，但可以通过boxsizing.htc使其IE6、IE7得到支持。
